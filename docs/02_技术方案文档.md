### 1.2 核心组件说明

#### 1.2.1 Planner (规划器)
负责面试流程的整体规划和决策，是Agent的"大脑"。

**核心职责:**
- 识别当前面试阶段（开场、自我介绍、项目深挖等）
- 决定下一步行动（提问、追问、转场、总结）
- 生成面试问题和反馈策略
- 协调各工具模块的调用

**实现方案:**
```python
# 伪代码示例
class InterviewPlanner:
    def plan_next_action(self, context: InterviewContext) -> Action:
        # 1. 分析当前对话状态
        current_phase = self.identify_phase(context)
        
        # 2. 评估候选人回答质量
        answer_quality = self.evaluate_answer(context.last_message)
        
        # 3. 决策下一步
        if current_phase == "project" and answer_quality == "vague":
            return Action(type="deep_dive", tool="project_analyzer")
        elif current_phase == "technical":
            return Action(type="ask_followup", tool="question_generator")
        # ...
```

#### 1.2.2 Tool层 (工具集)
封装各类外部能力，供Planner调用。

| 工具名称 | 功能描述 | 技术实现 |
|---------|---------|---------|
| ResumeParser | 解析Word简历，提取结构化信息 | python-docx + LLM |
| KnowledgeRetriever | 从知识库检索相关信息 | ChromaDB + Embedding |
| WebSearch | 联网搜索实时信息 | DuckDuckGo Search (免费) |
| ConversationSummarizer | 总结对话生成知识文档 | LLM |
| JobAnalyzer | 分析岗位要求，生成面试策略 | LLM + Prompt工程 |

#### 1.2.3 Memory (记忆系统)
管理Agent的各类记忆信息。

**三层记忆架构:**
1. **短期记忆**: 当前对话的上下文窗口
2. **工作记忆**: 本次面试的简历、岗位信息
3. **长期记忆**: 知识库中的历史积累

#### 2.3.3 LLM工厂模式实现

```python
from langchain_openai import ChatOpenAI
from langchain_anthropic import ChatAnthropic
from typing import Dict, Type

class LLMFactory:
    """LLM工厂类 - 支持多提供商动态切换"""
    
    _providers: Dict[str, Type] = {
        "openai": ChatOpenAI,
        "anthropic": ChatAnthropic,
        "deepseek": ChatOpenAI,  # DeepSeek兼容OpenAI接口
        "openrouter": ChatOpenAI,  # OpenRouter兼容OpenAI接口
        "custom": ChatOpenAI,  # 自定义OpenAI兼容接口
    }
    
    @classmethod
    def create_llm(cls, config: LLMConfig):
        """根据配置创建LLM实例"""
        provider_class = cls._providers.get(config.provider)
        if not provider_class:
            raise ValueError(f"不支持的提供商: {config.provider}")
        
        # 构建初始化参数
        init_params = {
            "model": config.model,
            "temperature": config.temperature,
            "max_tokens": config.max_tokens,
            "timeout": config.timeout,
        }
        
        # 不同提供商的参数处理
        if config.provider in ["openai", "deepseek", "openrouter", "custom"]:
            init_params["base_url"] = config.base_url
            init_params["api_key"] = config.api_key
        elif config.provider == "anthropic":
            init_params["anthropic_api_url"] = config.base_url
            init_params["anthropic_api_key"] = config.api_key
        
        return provider_class(**init_params)
    
    @classmethod
    def register_provider(cls, name: str, provider_class: Type):
        """注册新的提供商"""
        cls._providers[name] = provider_class
```

### 3.2 简历解析模块

#### 3.2.1 技术方案

```python
import docx
from langchain_openai import ChatOpenAI
from pydantic import BaseModel
from typing import List, Optional

class ResumeData(BaseModel):
    name: str
    phone: Optional[str]
    email: Optional[str]
    education: List[dict]
    experience: List[dict]
    skills: List[str]
    projects: List[dict]

class ResumeParser:
    def __init__(self):
        self.llm = ChatOpenAI(model="gpt-4o-mini")
    
    def parse_docx(self, file_path: str) -> ResumeData:
        # 1. 提取原始文本
        doc = docx.Document(file_path)
        raw_text = "\n".join([para.text for para in doc.paragraphs])
        
        # 2. 提取表格内容
        tables_text = []
        for table in doc.tables:
            for row in table.rows:
                row_text = [cell.text for cell in row.cells]
                tables_text.append(" | ".join(row_text))
        
        full_text = raw_text + "\n" + "\n".join(tables_text)
        
        # 3. LLM结构化提取
        structured_data = self.llm.with_structured_output(ResumeData).invoke(
            f"请从以下简历文本中提取结构化信息:\n{full_text}"
        )
        
        return structured_data
```

#### 3.3.2 文档自动总结（自动创建知识文档）

**功能说明:** 系统自动检测对话内容，在满足条件时自动生成Markdown格式的知识文档并保存到知识库。

**触发条件:**
1. 对话轮数达到阈值（默认5轮）
2. 内容变化检测（新信息占比>30%）
3. 用户主动触发总结
4. 面试阶段切换时

```python
from langchain.chains.summarize import load_summarize_chain
from langchain.docstore.document import Document
from datetime import datetime
import os

class KnowledgeSummarizer:
    """知识文档自动生成器 - 自动创建总结文档"""
    
    def __init__(self, llm_factory: LLMFactory, knowledge_base_path: str = "./knowledge_base"):
        self.llm_factory = llm_factory
        self.knowledge_base_path = knowledge_base_path
        self.summary_threshold = 5  # 对话轮数阈值
        
    def should_summarize(self, messages: list, last_summary_time: datetime = None) -> bool:
        """判断是否应该触发总结"""
        # 条件1: 对话轮数达到阈值
        if len(messages) >= self.summary_threshold:
            return True
        
        # 条件2: 距离上次总结时间超过阈值
        if last_summary_time:
            time_diff = datetime.now() - last_summary_time
            if time_diff.total_seconds() > 3600:  # 1小时
                return True
        
        return False
    
    def summarize_conversation(self, messages: list, user_id: str, session_id: str) -> dict:
        """
        总结对话内容并自动生成知识文档
        
        Returns:
            {
                "doc_id": "uuid",
                "file_path": "知识库/项目经历/xxx.md",
                "category": "项目经历",
                "title": "字节跳动推荐算法优化",
                "content": "...",
                "embedding_id": "..."
            }
        """
        # 1. 提取关键信息
        conversation_text = "\n".join([
            f"{m.role}: {m.content}" for m in messages
        ])
        
        # 2. 使用用户配置的LLM进行分类和总结
        llm = self.llm_factory.get_default_llm(user_id)
        
        # 3. 分类识别
        category = self._classify_content(conversation_text, llm)
        
        # 4. 生成总结
        summary_chain = load_summarize_chain(llm, chain_type="map_reduce")
        summary = summary_chain.run([Document(page_content=conversation_text)])
        
        # 5. 提取问答对
        qa_pairs = self._extract_qa_pairs(messages, llm)
        
        # 6. 生成标题
        title = self._generate_title(summary, llm)
        
        # 7. 生成Markdown文档内容
        doc_content = self._generate_markdown_doc(
            category=category,
            title=title,
            summary=summary,
            conversation_text=conversation_text,
            qa_pairs=qa_pairs,
            messages=messages
        )
        
        # 8. 自动保存到文件系统
        file_path = self._save_to_file(category, title, doc_content, user_id)
        
        # 9. 生成向量嵌入
        embedding = self._embed_document(doc_content)
        
        return {
            "category": category,
            "title": title,
            "content": doc_content,
            "file_path": file_path,
            "embedding": embedding,
            "source_session_id": session_id,
            "is_auto_generated": True
        }
    
    def _generate_markdown_doc(self, category: str, title: str, summary: str, 
                               conversation_text: str, qa_pairs: list, messages: list) -> str:
        """生成Markdown格式文档"""
        
        # 构建问答部分
        qa_section = "\n\n".join([
            f"**Q: {qa['question']}**\n\nA: {qa['answer']}"
            for qa in qa_pairs
        ])
        
        # 提取关键词
        keywords = self._extract_keywords(conversation_text)
        
        doc = f"""# {category} - {title}

> 📝 本文档由系统自动生成
> 📅 生成时间: {datetime.now().strftime('%Y-%m-%d %H:%M')}
> 🏷️ 分类: {category}

## 总结

{summary}

## 关键信息

{self._extract_key_points(messages)}

## 问答记录

{qa_section}

## 详细对话

<details>
<summary>展开查看完整对话</summary>

```
{conversation_text}
```

</details>

## 关键词

{', '.join([f'`{kw}`' for kw in keywords])}

---

*本文档由面试模拟Agent系统自动生成，如有错误请手动编辑修正*
"""
        return doc
    
    def _save_to_file(self, category: str, title: str, content: str, user_id: str) -> str:
        """自动保存文档到文件系统"""
        # 构建文件路径: knowledge_base/{user_id}/{category}/{title}.md
        safe_title = "".join(c for c in title if c.isalnum() or c in (' ', '-', '_')).strip()
        category_path = os.path.join(self.knowledge_base_path, user_id, category)
        os.makedirs(category_path, exist_ok=True)
        
        file_path = os.path.join(category_path, f"{safe_title}.md")
        
        # 如果文件已存在，添加序号
        counter = 1
        original_file_path = file_path
        while os.path.exists(file_path):
            file_path = original_file_path.replace('.md', f'_{counter}.md')
            counter += 1
        
        # 写入文件
        with open(file_path, 'w', encoding='utf-8') as f:
            f.write(content)
        
        return file_path
    
    def _classify_content(self, text: str, llm) -> str:
        """使用LLM分类内容"""
        prompt = f"""请分析以下面试对话内容，判断主要属于哪个类别：

可选类别：
- 个人介绍（自我介绍、个人优势等）
- 项目经历（具体项目描述、技术实现等）
- 技术知识（技术概念、原理、架构等）
- 行为面试（团队协作、冲突处理、领导力等）
- 岗位相关（目标公司、职位要求等）

对话内容：
{text[:1000]}...

请只返回类别名称，不要其他内容。"""
        
        response = llm.invoke(prompt)
        category = response.content.strip()
        
        # 映射到标准分类
        category_map = {
            "个人介绍": "个人介绍",
            "项目经历": "项目经历",
            "技术知识": "技术知识",
            "行为面试": "行为面试",
            "岗位相关": "岗位相关"
        }
        
        return category_map.get(category, "其他")
    
    def _extract_qa_pairs(self, messages: list, llm) -> list:
        """提取问答对"""
        qa_pairs = []
        for i in range(0, len(messages) - 1, 2):
            if messages[i].role == "assistant" and messages[i + 1].role == "user":
                qa_pairs.append({
                    "question": messages[i].content,
                    "answer": messages[i + 1].content
                })
        return qa_pairs[-5:]  # 只保留最近5个问答对
    
    def _generate_title(self, summary: str, llm) -> str:
        """生成文档标题"""
        prompt = f"""请根据以下总结内容，生成一个简洁的标题（不超过20个字）：

总结内容：
{summary}

标题："""
        response = llm.invoke(prompt)
        return response.content.strip()
    
    def _extract_key_points(self, messages: list) -> str:
        """提取关键信息点"""
        key_points = []
        for msg in messages:
            if msg.role == "user":
                # 提取包含数字、成果、技术栈的信息
                content = msg.content
                if any(kw in content for kw in ["%", "提升", "优化", "负责", "项目"]):
                    key_points.append(f"- {content[:100]}...")
        return "\n".join(key_points[:5])  # 最多5个要点
    
    def _extract_keywords(self, text: str) -> list:
        """提取关键词"""
        # 可以接入LLM或使用简单规则提取
        import re
        # 提取技术关键词（大写字母组合或特定模式）
        tech_patterns = re.findall(r'\b[A-Z][a-zA-Z]*\b', text)
        return list(set(tech_patterns))[:10]  # 去重后最多10个
    
    def _embed_document(self, content: str) -> list:
        """生成文档向量嵌入"""
        # 调用Embedding服务
        from langchain_openai import OpenAIEmbeddings
        embeddings = OpenAIEmbeddings()
        return embeddings.embed_query(content)


# 自动总结触发器 - 集成到Agent工作流中
class AutoSummaryTrigger:
    """自动总结触发器"""
    
    def __init__(self, summarizer: KnowledgeSummarizer):
        self.summarizer = summarizer
        self.session_message_count = {}  # 记录每个会话的消息数
        self.session_last_summary = {}   # 记录上次总结时间
    
    def on_message(self, session_id: str, user_id: str, messages: list):
        """每次消息时检查是否触发总结"""
        current_count = len(messages)
        last_count = self.session_message_count.get(session_id, 0)
        
        # 消息数增加达到阈值
        if current_count - last_count >= self.summarizer.summary_threshold:
            if self.summarizer.should_summarize(
                messages, 
                self.session_last_summary.get(session_id)
            ):
                # 触发自动总结
                result = self.summarizer.summarize_conversation(
                    messages, user_id, session_id
                )
                
                # 更新记录
                self.session_message_count[session_id] = current_count
                self.session_last_summary[session_id] = datetime.now()
                
                return result
        
        return None
```

```python
class DuckDuckGoSearchTool:
    """
    DuckDuckGo免费搜索工具
    特点：
    - 完全免费，无需API Key
    - 支持文本搜索、新闻搜索
    - 支持中文结果
    - 有速率限制（建议每秒1-2次请求）

    def __init__(self, region: str = "cn-zh", max_results: int = 5):
        self.region = region
        self.max_results = max_results
        self.ddgs = DDGS()
    
    def search(self, query: str, search_type: str = "text") -> List[Dict]:
        """
        执行搜索
        
        Args:
            query: 搜索关键词
            search_type: 搜索类型 (text/news)
        
        Returns:
            [{"title": "...", "href": "...", "body": "..."}]
        """
        try:
            if search_type == "text":
                results = self.ddgs.text(
                    keywords=query,
                    region=self.region,
                    safesearch="moderate",
                    timelimit=None,
                    max_results=self.max_results
                )
            elif search_type == "news":
                results = self.ddgs.news(
                    keywords=query,
                    region=self.region,
                    safesearch="moderate",
                    timelimit="w",  # 一周内
                    max_results=self.max_results
                )
            else:
                raise ValueError(f"不支持的搜索类型: {search_type}")
            
            return list(results)
            
        except Exception as e:
            print(f"DuckDuckGo搜索失败: {e}")
            return []
    
    def search_with_answer(self, query: str) -> Dict:
        """
        搜索并尝试直接获取答案（DuckDuckGo即时答案）
        """
        try:
            # DuckDuckGo有时会提供即时答案
            results = self.ddgs.answers(keywords=query)
            return {
                "has_answer": len(results) > 0,
                "answer": results[0] if results else None,
                "search_results": self.search(query)
            }
        except Exception as e:
            return {
                "has_answer": False,
                "answer": None,
                "search_results": self.search(query)
```

## 6. 安全与性能考虑

### 6.1 安全措施

#### 6.1.1 认证与授权
```python
# JWT配置
JWT_CONFIG = {
    "algorithm": "HS256",
    "access_token_expire_minutes": 30,
    "refresh_token_expire_days": 7
}

# 密码加密
from passlib.context import CryptContext
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")
```

#### 6.1.2 数据保护
- 敏感数据（简历、联系方式）AES-256加密存储
- 传输层强制TLS 1.3
- API请求速率限制（100次/分钟）

#### 6.1.3 输入验证
```python
from pydantic import BaseModel, validator

class ChatRequest(BaseModel):
    session_id: str
    message: str
    
    @validator('message')
    def validate_message(cls, v):
        if len(v) > 4000:
            raise ValueError('消息长度不能超过4000字符')
        return v.strip()
```

### 6.2 性能优化

#### 6.2.1 数据库优化
- 连接池配置（PostgreSQL: 20连接）
- 常用查询添加索引
- 消息表按时间分区

#### 6.2.2 缓存策略
```python
# Redis缓存配置
CACHE_CONFIG = {
```

#### 6.2.3 LLM调用优化

## 8. 关键技术难点与解决方案

### 8.1 Agent思考过程可视化

**难点:**
- LangGraph内部状态如何暴露给前端
- 流式输出的顺序和完整性保证
- 思考过程的格式化展示

**解决方案:**
- 自定义LangGraph回调处理器，捕获每个节点的输入输出
- 使用SSE协议保证有序传输
- 前端使用状态机管理不同事件类型的展示

### 8.2 简历解析准确性

**难点:**
- 简历格式多样，模板不统一
- 关键信息提取的准确性
- 表格内容的解析

**解决方案:**
- 结合规则提取和LLM理解
- 提供人工校对界面
- 建立常见简历模板库

### 8.3 知识库自动更新

**难点:**
- 何时触发总结（时机判断）
- 如何避免重复内容
- 新旧知识的整合

**解决方案:**
- 对话轮数阈值 + 内容变化检测
- 向量相似度去重
- 增量更新策略

### 8.4 面试场景真实性

**难点:**
- 不同岗位的面试侧重点
- 追问逻辑的合理性
- 反馈的专业性

**解决方案:**
- 建立岗位-面试策略映射表
- 使用Few-shot Prompting
- 引入领域专家审核机制
