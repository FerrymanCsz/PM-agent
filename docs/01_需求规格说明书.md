## 文档信息
- **版本**: v1.0
- **日期**: 2026-03-12
- **状态**: 调研阶段

---

## 1. 项目概述

### 1.1 项目背景
面向面试官的面试模拟Agent系统，通过AI技术模拟真实求职者，帮助面试官（用户）进行面试练习、评估面试问题质量、测试不同面试策略的效果。用户作为面试官出题，Agent作为面试者回答。

### 1.2 目标用户
- 准备面试他人的面试官
- 需要练习面试技巧的HR
- 想要测试面试问题的技术负责人
- 需要准备面试题库的企业

### 1.3 核心价值主张
- **真实性**: 模拟真实面试场景和流程
- **个性化**: 基于简历和岗位定制面试内容
- **智能化**: AI驱动的动态反馈和知识库构建
- **可追溯**: 完整的对话历史和知识积累

## 2. 功能需求规格

### 2.1 交互式聊天系统 (F-001)

#### 2.1.1 功能描述
实现类似传统大语言模型的聊天界面，用户作为面试官提问，Agent作为面试者回答。用户输入面试问题，Agent基于知识库和简历内容给出专业回答。

#### 2.1.2 详细需求

| 需求编号 | 需求描述 | 优先级 | 验收标准 |
|---------|---------|--------|---------|
| F-001-01 | 实现类ChatGPT的聊天界面 | P0 | 支持消息气泡、头像、时间戳显示 |
| F-001-02 | 支持实时对话流式输出 | P0 | 采用SSE技术，延迟<500ms |
| F-001-03 | 展示Agent思考过程 | P0 | 实时显示planner流程、工具调用 |
| F-001-04 | 对话历史完整记录 | P0 | 支持查看任意历史对话 |
| F-001-05 | 创建新对话功能 | P0 | 一键创建新面试会话 |
| F-001-06 | 对话会话管理 | P1 | 重命名、删除、搜索对话 |
| F-001-07 | 对话导出功能 | P2 | 支持导出为PDF/Markdown |

#### 2.1.3 界面原型要点
```
┌─────────────────────────────────────────────────────────────┐
│  [侧边栏]              │  [主聊天区域]                       │
│  ┌─────────────────┐   │  ┌─────────────────────────────┐   │
│  │ + 新面试        │   │  │ 系统: 面试开始               │   │
│  ├─────────────────┤   │  ├─────────────────────────────┤   │
│  │ 📋 历史对话      │   │  │ 用户(面试官): 你好，请先做个自我介绍 │   │
│  │ ├─ 模拟面试1     │   │  ├─────────────────────────────┤   │
│  │ ├─ 模拟面试2     │   │  │ Agent(面试者): 您好！我是张三...     │   │
│  │ └─ 模拟面试3     │   │  │ [思考过程 ▼]                 │   │
│  ├─────────────────┤   │  │ 正在检索知识库...            │   │
│  │ 📁 知识库        │   │  │ 正在组织回答...              │   │
│  │ 📄 简历管理      │   │  ├─────────────────────────────┤   │
│  │ 🔍 岗位设置      │   │  │ 用户(面试官): 能详细说说你在XX项目中的角色吗？ │   │
│  └─────────────────┘   │  ├─────────────────────────────┤   │
│                        │  │ Agent(面试者): 在XX项目中，我主要负责... │   │
│                        │  ├─────────────────────────────┤   │
│                        │  │ [输入框] [发送]              │   │
│                        │  └─────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 简历上传与解析模块 (F-002)

#### 2.2.1 功能描述
支持上传Word格式简历文件，实现简历内容的自动解析与结构化存储。

#### 2.2.2 详细需求

| 需求编号 | 需求描述 | 优先级 | 验收标准 |
|---------|---------|--------|---------|
| F-002-01 | 支持.docx格式上传 | P0 | 支持拖拽上传，文件大小<10MB |
| F-002-02 | 简历内容解析 | P0 | 提取文本、表格、结构化信息 |
| F-002-03 | 关键信息识别 | P0 | 姓名、联系方式、教育背景、工作经历、技能 |
| F-002-04 | 解析结果编辑 | P1 | 支持人工校对和修正 |
| F-002-05 | 多简历管理 | P1 | 支持上传多份简历版本 |
| F-002-06 | 简历与对话关联 | P0 | 解析结果作为对话上下文 |

```json
{
  "basic_info": {

  "education": [{
    "school": "学校",
    "major": "专业",
    "degree": "学位",
    "period": "时间段"
  }],
  "experience": [{
    "company": "公司",
    "position": "职位",
    "period": "时间段",
    "description": "工作描述"
  }],
  "skills": ["技能1", "技能2"],
  "projects": [{
    "name": "项目名称",
    "description": "项目描述",
    "technologies": ["技术栈"]
  }]
}
```

### 2.3 智能知识库系统 (F-003)

#### 2.3.1 功能描述
初始状态为空，支持动态构建与更新，实现对话内容的自动/手动总结功能。

#### 2.3.2 详细需求

| 需求编号 | 需求描述 | 优先级 | 验收标准 |
|---------|---------|--------|---------|
| F-003-01 | 知识库初始为空 | P0 | 新用户从零开始构建 |
| F-003-02 | 对话内容自动总结 | P0 | AI自动提取关键信息生成文档 |
| F-003-03 | 手动创建/编辑文档 | P0 | 支持Markdown格式编辑 |
| F-003-04 | 文档分类管理 | P0 | 按知识板块/项目板块分类 |
| F-003-05 | 文档树形结构 | P1 | 支持文件夹、子文件夹 |
| F-003-06 | 文档拖拽排序 | P1 | 支持拖动调整文档位置 |
| F-003-07 | 智能匹配关联 | P0 | 面试时自动关联相关知识 |
| F-003-08 | 文档版本历史 | P2 | 支持查看和恢复历史版本 |

#### 2.3.3 知识库分类体系

### 2.4 联网搜索能力 (F-004)

#### 2.4.1 功能描述
集成联网搜索工具，确保Agent的回复结合实时搜索信息与自身模型能力。

#### 2.4.2 详细需求

| 需求编号 | 需求描述 | 优先级 | 验收标准 |
|---------|---------|--------|---------|
| F-004-01 | 集成搜索API | P0 | 支持Tavily/Serper等搜索服务 |
| F-004-02 | 搜索结果筛选 | P1 | 按时间、来源、相关性筛选 |
| F-004-03 | 搜索内容整合 | P0 | 智能整合搜索结果到回复中 |
| F-004-04 | 搜索历史记录 | P2 | 记录搜索查询和结果 |
| F-004-05 | 搜索触发策略 | P1 | 自动判断何时需要搜索 |

#### 2.4.3 搜索应用场景
- 目标公司最新动态查询
- 行业技术趋势了解
- 面试问题参考答案搜索
- 岗位薪资水平调研

### 2.5 岗位定制化功能 (F-005)

#### 2.5.1 功能描述
支持输入目标岗位信息，根据岗位信息调整面试模拟的反馈内容与流程。

#### 2.5.2 详细需求

| 需求编号 | 需求描述 | 优先级 | 验收标准 |
|---------|---------|--------|---------|
| F-005-01 | 岗位信息录入 | P0 | 公司、岗位、部门、职级 |
| F-005-02 | JD解析 | P1 | 自动提取岗位要求和技能 |
| F-005-03 | 面试策略调整 | P0 | 根据岗位调整问题类型和难度 |
| F-005-04 | 多岗位配置 | P1 | 保存多个岗位配置 |
| F-005-05 | 岗位模板库 | P2 | 预设常见岗位模板 |

#### 2.5.3 岗位信息结构
```json
{
  "company": "公司名称",
  "position": "岗位名称",
  "department": "部门",
  "level": "职级",
  "job_description": "岗位描述",
  "requirements": ["要求1", "要求2"],
  "skills": ["技能1", "技能2"],
  "interview_focus": ["技术深度", "项目经验", "团队协作"]
}
```

### 2.6 面试场景覆盖 (F-006)

#### 2.6.1 功能描述
全面覆盖真实面试场景中的各类模块。

#### 2.6.2 面试场景模块

| 场景模块 | 描述 | 优先级 | 特点 |
|---------|------|--------|------|
| 自我介绍 | 开场白、个人背景介绍 | P0 | 基于简历自动生成建议 |
| 项目经历 | 项目介绍、深挖细节 | P0 | STAR法则引导、技术细节追问 |
| 技术问题 | 专业知识考察 | P0 | 根据岗位定制技术栈问题 |
| 行为面试 | 软技能、情境问题 | P0 | 宝洁八大问、亚马逊领导力准则 |
| 薪资谈判 | 薪酬期望沟通 | P1 | 策略指导、话术建议 |
| 职业规划 | 长期发展探讨 | P1 | 结合行业趋势分析 |
| 反问环节 | 向面试官提问 | P1 | 智能推荐高质量问题 |
| 压力面试 | 高压情境模拟 | P2 | 挑战性问题、打断技巧 |

#### 2.6.3 面试流程设计

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  开场寒暄    │ -> │  自我介绍    │ -> │  项目经历    │
└─────────────┘    └─────────────┘    └──────┬──────┘
                                             │
┌─────────────┐    ┌─────────────┐    ┌──────▼──────┐
│  反问环节    │ <- │  薪资/规划   │ <- │  技术问题    │
└─────────────┘    └─────────────┘    └─────────────┘
```

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 要求 | 说明 |
|-----|------|------|
| 首字响应时间 | < 2秒 | 用户发送消息后首字出现时间 |
| 流式输出延迟 | < 500ms | 每个token的传输延迟 |
| 页面加载时间 | < 3秒 | 首屏完全加载时间 |
| 并发用户数 | > 100 | 系统支持的并发会话数 |

### 3.2 可用性需求

| 指标 | 要求 | 说明 |
|-----|------|------|
| 系统可用性 | 99.5% | 月度可用时间比例 |
| 数据持久化 | 100% | 对话数据不丢失 |
| 故障恢复 | < 5分钟 | 自动故障恢复时间 |

### 3.3 安全需求

| 需求 | 说明 |
|-----|------|
| 数据加密 | 传输层TLS 1.3，存储层AES-256 |
| 访问控制 | 基于JWT的身份认证 |
| 隐私保护 | 简历等敏感数据脱敏存储 |
| API安全 | 速率限制、请求签名验证 |

### 3.4 兼容性需求

| 平台 | 支持版本 |
|-----|---------|
| Chrome | 最新2个版本 |
| Firefox | 最新2个版本 |
| Safari | 最新2个版本 |
| Edge | 最新2个版本 |
| 移动端 | iOS 14+, Android 10+ |

## 4. 用户界面需求

### 4.1 界面风格
- **设计语言**: 简洁现代、专业商务
- **主色调**: 深蓝色(#1a365d) + 白色背景
- **字体**: 系统默认字体栈，中文优先PingFang SC/Microsoft YaHei
- **布局**: 侧边栏 + 主内容区的经典布局

### 4.2 响应式设计
| 断点 | 布局调整 |
|-----|---------|
| Desktop (>1280px) | 完整三栏布局 |
| Tablet (768-1280px) | 侧边栏可折叠 |
| Mobile (<768px) | 单栏布局，底部导航 |

### 4.3 交互规范
- 所有操作提供即时反馈
- 加载状态使用骨架屏或旋转动画
- 错误提示使用Toast通知
- 确认操作使用Dialog弹窗
